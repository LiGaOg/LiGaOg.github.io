处理器的高速缓存

为什么需要Cache

  • Spatial Locality: CPU访问了内存的一个地址, 它还会访问这个地址周围的地址.

  • Temporal Locality: CPU再某个时间访问了一个地址, 它很有可能在最近还访问.

  • Cache一般集成在CPU内部的SRAM中, 如果cache命中, 可以以更少的时钟周期拿到数据.

衡量Cache性能的指标

  • AMAT (Average Memory Access Time)
    • AMAT = HIT TIME + MISS RATE * MISS PENALTY
  • 因此, 有三种方法优化Cache的性能:
    • 减少Hit time.
      • 用更快更小的cache.
    • 减少Miss rate.
      • 用更大的cache, cache的策略选的好一点.
    • 减少Miss penalty.
      • 减少main memory的访问速度.

Cache的工作原理

Direct Mapped Cache

首先, Cache存储了很多cache block, 每一个block有一个大小b, 假设这个cache的大小是C, 那么就一共有C / b个cache block.

对于一个物理地址:

  • 最后的log(b)位可以看做这个地址在cache block中的offset.
  • 之后再往前的log(C / b)为可以作为index, 定位你访问的数据在哪个cache block.
  • 再往前的所有Bits可以看做一个Tag, 每一个cache block也有一个Tag.
    • 如果这两个Tag相同, 那么cache hit.
    • 如果这两个Tag不同, 那么cache miss, 系统会从main memory中读取相应的block, 写入对应的cache block, 然后写入Tag.
  • 每个cache block还有一个valid bit, 标识这个block中是否存储了从main memory中带来的数据, 并且这个数据是否有效.

N-way Set Associative Cache

现在, 我把大小为C的Cache分成了若干个Set, 每一个Set中有N个cache blocks, 每一个cache block的大小是b.

对于一个物理地址:

  • 最后的log(b)还是cache block中的offset.
  • 再往前的log(C/N)可以作为index, 定位你的数据在哪一个Set.
  • 再往前的所有Bits可以看做一个Tag, 每一个cache block对应一个Tag.
    • 定位到Set之后, 访问的物理地址的Tag需要和Set中的每一个cache block的Tag对比, 如果有一个一样, 那么就cache hit.
    • 如果都不一样, 那么就cache miss, 系统会从main memory中把对应的block读进Set中, 至于放到Set中的哪一个block, 这个就取决于cache replacement policy.
  • 同样, 每个cache block也有valid bit.

Fully Associative Cache

现在Cache存储了很多的block, 每一个block大小为b, cache的大小为C.

对于一个物理地址:

  • 最后log(b)标识cache block中的offset.
  • 然后, 剩下的所有bits全部是Tag, 物理地址的Tag需要和所有C / b个cache block的Tag做比较:
    • 如果有一个相同, 那么就cache hit.
    • 如果都不同, 那么cache miss, CPU把block从内存中读出来后, 会按照cache replacement policy放到一个cache block中.
  • 同样, 每个cache block也有valid bit.

三种Cache架构的联系

假设Cache的大小是C, cache block的大小是b.

  • Direct Mapped Cache相当于1-way Set Associative Cache, 每个Set中cache block的个数是C / b.
  • Fully Associative Cache相当于C / b-way Set Associative Cache, 一共就1个Set.

Cache Miss的来源

  • Compulsory Miss: 第一次访问肯定Cache Miss, 因为Cache里面就没有东西.
  • Capacity Miss: Cache已经放满了, 再来一个Cache中没有的肯定Miss.
  • Conflict: Conflict的精准定义是, 如果这个Miss在Fully Associative Cache中不会发生, 那么这个Miss就是Conflict Miss.
    • Conflict Miss和Cache replacement policy相关.
