我从本科三年级开始学习Linux，并使用neovim作为编辑器。在我眼中，vim相当于一种高效文本编辑的协议。不同的软件都可以实现vim的编辑模式，例如vscode，IDEA的vim插件。借助这个协议，用户可以完全脱离鼠标，快速完成某些文本操作。同时，配合一些vim插件，vim还能够具有更强大的功能。这个文章主要用来记录我个人工作时常用的vim命令、插件以及相关配置，会持续更新。

Part I: vim的基本使用

首先是vim的一些常识，vim有三种模式：

  • Normal：可以用命令进行文本编辑。
  • Insert：用来输入文本。
  • Visual：用来选择文本。

其中这三种模式可以用<Esc>，i，v进行互相切换，其中大部分要介绍的命令，如果没做特殊说明的话，都是在Normal和Visual模式下。

从Normal模式转到Insert模式有很多种方法：

  • i：光标移动到所在字符之前。
  • I：光标移动到这一行最靠前的文本的开头。
  • a：光标移动到所在字符之后。
  • A：光标移动到这一行最靠后的文本的后面。
  • o：光标移动到所在行的下一行。
  • O：光标移动到所在行的上一行。

从Normal模式转到Visual模式也有很多方法：

  • v：一般的选择模式。
  • V：行选择。
  • ctrl + v：块选择。

移动光标

vim中的上下左右移动光标是h, j, k, l。一般使用的时候，需要快速上下移动，这时候可以用5j, 5k，一次上下移动5行。这里我配置了一个快捷键，可以用J,K进行快速移动。

快速移动到行首行尾是^和$，这里被我配置成了H和L，因为如果要快速在一行内移动，有其他的命令。

如果要快速在一行内移动 ，可以用w和b命令。w是以word为单位向后移动，b是以word为单位向前移动。

在一行内移动还有一个命令是f，先摁f，然后加个字符，可以快速跳转到本行中光标后面第一次出现这个字符的地方。

快速跳到第一行可以用gg，快速到最后一行可以用G，跳转到某一个具体行可以用:linenumber。

vim中有一个叫做jumplists的机制，当你执行了一个“跳转命令”时（跳转命令可以包括gg, gG， 搜索等，可以用:h jump-motions查看哪些命令算跳转），vim会把执行命令前光标的位置记录下来。用ctrl + o可以返回上一个记录的位置，ctrl + i可以去下一个记录的位置。

复制粘贴和撤销

复制的命令是y，一般的模式就是选中若干文本然后摁y，就能复制文本了。

yy可以快速复制一行文本。

假设有一些文本是[name]：

  • 如果要复制name，可以用yi[。
  • 如果要复制[name]，可以用ya[。

p是粘贴文本，如果复制的是一行/多行内容，粘贴时就会粘贴到下一行。

如果要和系统的剪切板同步，可以用"+y，粘贴到寄存器+中，这个+就是系统剪切板。粘贴可以用"+p。一般的命令是"<小写字母，代表寄存器名字>y/p。

如果要追加复制，假设现在内容复制到寄存器a中，我想要另外选择一段内容追加到a中，那么就用命令"Ay，其中a变成了大写。

撤销命令是u，取消撤销就是ctrl + u。

删除文本

删除字符命令是x和s，x删除后不进入Insert模式，而s进入。

删除文本的命令是d和c，它们的行为和y一样，不同的是，d删除文本后会停留在Normal模式，c删除后会进入Insert模式。

注意vim中删除文本都是剪切命令。

搜索替换

/和?都能在全文进行查找，在neovim中应该是效果相同。

n跳转到下一个查找结果，N跳转到上一个查找结果。

在全文中查找word1，并替换为word2，替换前询问要不要替换：%s/word1/word2/gc

选择文本

v的行为和c, d, y类似，可以通过yiw, yaw这种命令选择特定的文本。

v配合%可以快速选中某个函数/类，当光标在函数名的时候，首先用V选中这一行，然后选中{，然后用%就可以自动选中整个函数/类体。这个可以配合代码折叠。

分屏移动

经过配置后，分屏命令是sh, sj, sk, sl，在屏幕之间移动的命令是<space>h, <space>j, <space>k, <space>l。

折叠代码

首先折叠代码有很多种方式，这里我设置的方式是手动折叠：:set foldmethod=manual

折叠代码，就是先选中一段代码，然后用zf，打开折叠就是用zo

Tab标签页

Tab标签页主要和coc.nvim中的coc-explorer一起使用，当用coc-explorer打开一个文件时，这个文件是在一个新的Tab中打开。

zj切换到前一个Tab，zl切换到后一个Tab。

Normal指令

Normal指令可以将一系列vim命令用在选中的每一行上，使用方法为：

  • 首先选中若干行。
  • 然后输入:normal，然后后面跟着vim指令。
  • 之后摁下回车。

键盘宏

键盘宏：用于封装一系列vim操作，使用方法如下：

  • 摁q加上一个字母，这个字母是键盘宏的名称，然后进行vim操作。

  • 再摁q结束录制。

  • 之后，只要摁@字母，就能执行对应的键盘宏操作。

Normal指令可以和键盘宏一起使用，先用一行录制好键盘宏，然后Normal指令直接使用键盘宏即可。

代码格式化

gg=G可以将所有代码缩进格式化。

Part II: vim插件使用与管理

neovim配置文件在~/.config/nvim/init.vim

vim-plug

vim插件管理使用的是vim-plug，官方仓库是：vim-plug的官方仓库。

只需要把官方仓库中的plug.vim放到nvim/autoload文件夹中即可，autoload文件夹需要自己创建。

  • 每次添加插件后，用:PlugInstall命令直接下载即可。

  • 如果要更新vim-plug，需要用:PlugUpgrade。

  • 如果要更新所有插件，需要用:PlugUpdate

coc.nvim

自动补全插件使用的是coc.nvim，官方仓库是：coc.nvim的官方仓库，注意在使用之前，需要下载Node.js与npm。

  • coc.nvim中有自动检测插件更新的配置，所以不用手动执行命令。

Part III: 一些好用的插件

vim-commentary

能够快速注释选中的文本。

开箱即用，不用做特殊配置。

选中文本后使用gc即可注释，再使用gc可以解除注释。

vim-highlightedyank

在注释时会高亮复制到的文本。

开箱即用，不用做特殊配置。

vim-surround

能够高效处理环绕符号，例如括号等。

开箱即用，不用做特殊配置。

  • cs<字符1><字符2>：将原来用字符1环绕的文本用字符2环绕。
    • 如果字符1是<，会出现提示是否用HTML标签环绕，如果不用直接摁>即可。
  • ds<字符>：删除字符环绕。
  • ys：为某段文本添加环绕，例如ysiw"，可以给文本添加""。
    • 如果要用HTML标签环绕，可以用ysiw<，然后输入HTML标签内容。

vim-wildfire

能够智能选择文本单位，例如环绕符之间的文本，函数，类等。

开箱即用，不用做特殊配置。

连续摁<Enter>即可选择不同层级的文本单位，可以快速选择某一个函数/类。

vim-airline

相当于vim的状态栏，能够显示当前光标的位置信息，文件编码方式，当前的git分支等信息。

配置见Appendix。

vim-devicons

给不同后缀文件名添加图标，可以在Tab/vim-airline中看到图标。

开箱即用。

Vista.vim

显示函数/类/结构体等大纲信息，用ss即可调出。

开箱即用。

coc.nvim

neovim下的自动补全相关的插件，配置见Appendix。

coc.nvim给出了几个基本的命令：

  • ctrl+d：以列表的形式展示所有的语法错误。在列表中摁<Enter>可快速跳转。
  • [p：跳转到下一个语法错误。
  • ]p：跳转到上一个语法错误。
  • sd：显示文档。
  • gd：去一个符号的定义处。
  • gr：去一个符号的引用处，会以列表的形式展示所有的引用，<Enter>可跳转。
  • gi：去一个符号的实现处。

Part IV: coc.nvim插件

coc.nvim配置文件在~/.config/nvim/coc-settings.json

coc-explorer

一个文件浏览器，配置见Appendix，注意配置文件中使用了一些图标，需要提前下载Nerdfont。

coc-snippets

这个插件可以快速用snippets进行模板级补全，模板一般有一些参数。

  • ctrl + n：跳转到下一个补全参数的地方（在SNIP Insert模式下）
  • ctrl + p：跳转到上一个补全参数的地方

以上同样适用于函数补全。

其余补全与具体语言有关，可以参考：https://github.com/neoclide/coc.nvim/wiki/Using-coc-extensions
